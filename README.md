# Slukhach 👂

Telegram-бот, который приглушает основной (передний) голос в записи и усиливает
задний фон — чтобы можно было расслышать, что происходит на фоне (например,
разговор людей позади говорящего).

## Два режима обработки

Главный нюанс: «передний голос» и «задний фон» можно понимать двумя разными
способами, и для каждого нужен свой инструмент. Режим выбирается через
`PROCESSING_MODE`.

### `enhance` (по умолчанию) — для «людей за стенкой»

Когда и спереди, и сзади **говорят люди**, отличает их не тембр, а **громкость**:
близкий голос — громкий и чёткий, дальний — тихий и приглушённый. Поэтому здесь
работает не разделение источников, а **динамическая обработка**:

```
Аудио → [ ffmpeg filtergraph ]
        highpass    → убираем низкочастотный гул
        afftdn      → спектральное шумоподавление
        acompressor → сжимаем громкий передний голос
        dynaudnorm  → поднимаем тихие участки (там и есть задний разговор)
        volume      → итоговый makeup-gain
        alimiter    → защита от клиппинга
      → OGG/Opus → ответ
```

Быстро, без скачивания моделей. Параметры тюнятся через переменные окружения.

### `separate` — для музыки/песен

Demucs (модель Meta) разделяет запись на «голос» (`vocals`) и инструменты. Это
полезно, чтобы приглушить **поющего** и оставить аккомпанемент. Важно: Demucs
считает **любую речь** «вокалом», поэтому для записей «голос на фоне другого
голоса» этот режим не подходит — он приглушит и задний разговор тоже.

## Стек

- **Python 3.10+**
- **aiogram 3** — асинхронный фреймворк для Telegram-ботов
- **ffmpeg** — декодирование/кодирование и вся обработка в режиме `enhance`
  (compressor, dynaudnorm, afftdn, alimiter, EQ)
- **Demucs + PyTorch / torchaudio** — разделение источников в режиме `separate`
- **soundfile / numpy** — работа с WAV и сэмплами (режим `separate`)

## Установка

1. Установите [ffmpeg](https://ffmpeg.org/) и убедитесь, что он в `PATH`:

```bash
# Windows (winget)
winget install Gyan.FFmpeg
# Debian/Ubuntu
sudo apt install ffmpeg
# macOS
brew install ffmpeg
```

2. Создайте виртуальное окружение и установите зависимости:

```bash
python -m venv .venv
# Windows
.venv\Scripts\activate
# Linux/macOS
source .venv/bin/activate

pip install -r requirements.txt
```

3. Создайте файл `.env` на основе примера и впишите токен от
   [@BotFather](https://t.me/BotFather):

```bash
cp .env.example .env
```

## Запуск

```bash
python -m slukhach
```

Режим `enhance` работает сразу. В режиме `separate` при первом запуске Demucs
скачает веса модели (несколько сотен МБ).

## Настройка

Все параметры задаются через переменные окружения (см. `.env.example`):

| Переменная                | По умолчанию | Описание                                              |
| ------------------------- | ------------ | ----------------------------------------------------- |
| `BOT_TOKEN`               | —            | Токен бота от @BotFather (обязательно)                |
| `PROCESSING_MODE`         | `enhance`    | `enhance` (люди за стенкой) / `separate` (музыка)     |
| `MAX_FILE_SIZE_MB`        | `20`         | Максимальный размер входного файла                    |
| **enhance**               |              |                                                       |
| `HIGHPASS_HZ`             | `100`        | Срез низких частот (гул), Гц                          |
| `COMPRESSOR_THRESHOLD_DB` | `-28`        | Порог, выше которого сжимается громкий передний голос |
| `COMPRESSOR_RATIO`        | `8`          | Степень сжатия (больше = сильнее давит передний голос)|
| `MAKEUP_GAIN_DB`          | `6`          | Выходное усиление после компрессии (дБ)               |
| `DYNAUDNORM_MAX_GAIN`     | `20`         | Макс. подъём тихих участков (1 = выкл.)               |
| `NOISE_REDUCTION_DB`      | `6`          | Сила шумоподавления, дБ (0 = выкл.)                   |
| **separate**              |              |                                                       |
| `VOCAL_GAIN_DB`           | `-12`        | Насколько приглушить голос (дБ)                       |
| `BACKGROUND_GAIN_DB`      | `8`          | Насколько усилить фон (дБ)                             |
| `DEMUCS_MODEL`            | `htdemucs`   | Имя модели Demucs                                     |
| `DEVICE`                  | `auto`       | `auto` / `cpu` / `cuda`                               |

### Если фон всё ещё слишком тихий

В режиме `enhance` усиливайте эффект так: увеличивайте `DYNAUDNORM_MAX_GAIN`
(например, до `30`), повышайте `COMPRESSOR_RATIO` (до `12–20`) и понижайте
`COMPRESSOR_THRESHOLD_DB` (например, до `-35`). Если появляется шум — поднимите
`NOISE_REDUCTION_DB`.

## Структура проекта

```
slukhach/
├── __main__.py        # точка входа (python -m slukhach), выбор режима
├── bot.py             # хендлеры aiogram
├── config.py          # загрузка и валидация настроек
└── audio/
    ├── processor.py   # общий интерфейс Processor (стратегия обработки)
    ├── io_utils.py    # декодирование/кодирование/фильтры через ffmpeg
    ├── enhancer.py    # режим enhance: динамическая обработка (ffmpeg)
    ├── separator.py   # режим separate: обёртка над Demucs
    ├── remixer.py     # режим separate: микширование стемов с усилением
    └── pipeline.py    # режим separate: оркестрация разделения
```

## Замечания по производительности

- Режим `enhance` быстрый — это один проход ffmpeg, без нейросети.
- В режиме `separate` на CPU обработка минутного аудио может занимать заметное
  время; для скорости используйте GPU (`DEVICE=cuda` при наличии CUDA-сборки PyTorch).
- Лимит размера файла (`MAX_FILE_SIZE_MB`) защищает от слишком долгих задач.
